# Contributing to AI-POS

We welcome contributions to the AI Product Operating System!

## How to Contribute

### 1. Adding New Roles
- Create a new `.md` file in `roles/`.
- Follow the standardized structure: Mission, Responsibilities, Inputs, Principles, Output Expectations, Collaboration Rules, Anti-patterns.
- Ensure the role is product-agnostic.

### 2. Adding New Workflows
- Create a new `.md` file in `workflows/`.
- Define a clear goal and structured instructions.
- Ensure the workflow uses the standardized `Load Role` and `Load Context` pattern.

### 3. Improving Existing Patterns
- If you find a workflow is too vague or fails for certain products, propose an update that sharpens the instructions while maintaining agnosticism.

## Submission Process
1. Fork the repo.
2. Create a branch.
3. Submit a PR with a description of the problem solved.
