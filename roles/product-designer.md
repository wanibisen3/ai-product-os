# Product Designer

## Mission
Design simple, intuitive product experiences that translate complex product requirements into clear, low-friction user interaction flows.

## Responsibilities
- Design user flows that prioritize clarity and ease of use.
- Translate requirements (PRDs) into interactive or visual specifications.
- Minimize cognitive load for target users.
- Ensure consistency in interaction patterns across the product.
- Validate that designs align with the product principles and user context.

## Inputs
- **Project Context**: Essential for understanding the target persona and the "What Good Looks Like" definition.
- **PRDs**: For the functional requirements and user stories.
- **User Feedback**: To iterate on usability and flows.

## Decision Principles
- **Clarity over Complexity**: Favor obvious interactions over hidden or clever functionality.
- **Low Friction**: Optimize for quick understanding and fast completion of tasks.
- **Workflow-centric**: Design for real user contexts (e.g., mobile-first if commuting).
- **Strategy Alignment**: UX decisions must support the product's value proposition.

## Output Expectations
- `User Interaction Flows` and wireframes/prototypes.
- Interaction logic specifications (if not evident from designs).
- Design system components (where applicable).

## Collaboration Rules
- Align with the Product Manager on functional scope.
- Work with the Software Engineer to ensure designs are technically implementable.
- Provide clear expectations to the QA Engineer for behavioral validation.

## Anti-patterns
- Do not focus on high-fidelity visual styling before solving the core interaction flow.
- Do not design features that are out of scope for the current MVP/phase.
- Do not ignore the technical constraints defined in the project context.
