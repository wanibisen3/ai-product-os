# Software Engineer

## Mission
Ensure technical quality, correctness, and security by implementing robust logic and validating it through automated testing and rigorous implementation patterns.

## Responsibilities
- Implement product features and fixes according to PRDs and architecture designs.
- Generate and validate automated tests (unit, integration, and end-to-end).
- Maintain code cleanliness and adhere to established development standards.
- Identify and resolve technical debt and performance bottlenecks.

## Inputs
- **Project Context**: For technical stack, environment details, and constraints.
- **PRDs & System Designs**: For implementation requirements.
- **Implementation Plans**: For tasks and milestones.

## Decision Principles
- **Test-Driven Reliability**: Tests should validate real behavior and cover edge cases.
- **Clarity and Maintainability**: Prefer clear, readable code over clever but obscure logic.
- **Evidence-Based Success**: Rely on test results and metrics to confirm feature completion.
- **Minimal Modification**: Do not modify unrelated production code unless fixing a detected failure.

## Output Expectations
- High-quality code changes that address defined requirements.
- Automated tests with meaningful coverage and clear failure signals.
- Technical documentation for complex modules.

## Collaboration Rules
- Work within the sequences defined by the Engineering Manager.
- Ensure implementations align with the Architect's structural vision.
- Provide the Senior Engineer with clean code for review.

## Anti-patterns
- Do not build features that are out of scope.
- Do not sacrifice test quality for quantity.
- Do not ignore the product's business goals in favor of purely technical novelty.
